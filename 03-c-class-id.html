class="message" Les classes sont des attributs qui seront principalement utiles avec CSS et Javascript, elles permettent d'identifier un type de balises. Par exemple vous pouvez créer une class "important" et mettre celle ci à toute les balises "p" que vous voulez, pour ensuite grâce au CSS les faire apparaitre en gros et rouge.

id="mainArticle" les id peuvent servir de la même façon au CSS et Javascript. Cependant à cela s'ajoute de multiples utilités comme les ancres qui permettent de créer des liens vers un point précis d'une page. Ou encore de lier un label à un champ ce qui sera vu avec les formulaires.
En contrepartie une règle s'ajoute à l'utilisation des "id", si on peut avoir autant d'élément que l'on veut avec une même class, un id ne peut se retrouver qu'UNE seule et unique fois dans une même page. Evidement vous pouvez le réutiliser dans une page différente.

Pour ce qui est des noms que vous leur donner, il ne faut pas d'espace, faites attention si vous utilisez des tirets "-" ou des underscore "_" si en html c'est totalement géré, certains langages pourraient poser problème.
Les noms ne peuvent pas commencer par des chiffres, mais peuvent en contenir.
Une possibilité courrante de nommer des classes, id ou plus tard des variables est ce qu'on appelle le "Camel Case", écrire les différents mots sans espace avec une majuscule à chaque mot. (On parle de lower camel case si le premier mot n'a pas de majuscule ou upper camel case si le premier mot commence en majuscule).

  • nomEnLowerCamelCase
  • nomDeClasse
  • NomEnUpperCamelCase
  • NomDeClasse

Attention, dans les exemples si dessus "nomDeClasse" et "NomDeClasse" sont bien des classes différentes. On dit que html est "sensible à la caste". Cela signifie qu'il fait bien la différence entre minuscule et majuscule.
